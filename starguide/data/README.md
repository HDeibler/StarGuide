# Bundled astronomical data

These JSON files are generated by [`tools/build_catalog.py`](../../tools/build_catalog.py)
from public datasets. The StarGuide *code* is MIT; this *data* keeps the licenses
of its upstream sources, credited below.

| File | What | Rows |
|------|------|------|
| `hip_catalog.json` | `{HIP: {ra, dec, mag}}`, stars to magnitude 7.5 | 25,713 |
| `hip_names.json` | `{HIP: proper name}` | 450 |
| `constellation_data.json` | `{Constellation: [[HIP, HIP], …]}`, 88 IAU figures | 88 |

## Sources & licenses

- **Star positions, magnitudes, and proper names** — the
  [HYG database](https://github.com/astronexus/HYG-Database) (Astronomy Nexus),
  built from Hipparcos, Yale Bright Star, and Gliese catalogs. Licensed
  **CC BY-SA 2.5**.
- **Constellation stick figures** (which stars connect) — the "western" sky
  culture `constellationship.fab` from
  [Stellarium](https://github.com/Stellarium/stellarium) (v0.20.0). Stellarium is
  **GPL-2.0+**; the figure data is reference/factual.

To regenerate (re-downloads the sources into `tools/.cache/`):

```bash
python tools/build_catalog.py
```

If you need the bundled data under a different license, swap the sources in the
build script — the loaders in `starguide/astro.py` only assume the HIP-keyed JSON
shapes above.
